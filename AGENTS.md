# Instructions for Future AI Coding Agents

This repository is for a beginner-friendly neuroscience outreach project called "Visualizing the Brain".

## Audience

- Students are about 13-14 years old.
- Explanations should be clear, concrete, and friendly.
- Avoid university-level assumptions unless they are explained carefully.

## Priorities

- Prioritize clarity over cleverness.
- Keep dependencies minimal.
- Keep code cells short.
- Use simple variable names.
- Add helpful comments to notebook code.
- Prefer robust, stable nilearn functionality.
- Make notebooks executable from top to bottom in Google Colab.

## Scope Boundaries

- Do not add decoding, machine learning, or advanced fMRI preprocessing yet.
- Do not add large data files to the repository.
- Use nilearn built-in templates or small built-in examples where possible.
- Avoid workflows that require long downloads or high memory use.

## Notebook Style

- Write explanations for 13/14-year-old students.
- Use short markdown sections with plain language.
- Code should be readable before it is compact.
- Each code cell should do one small thing when possible.
- Google Colab compatibility is required.

## Repository Hygiene

- Keep generated data and large outputs out of git.
- Keep helper functions small and easy to inspect.
- If adding new dependencies, explain why they are needed.
